# easy-claw v0.3 设计：交互式 CLI 流式输出与工具调用显示

日期：2026-05-03

## 决策

v0.3 只增强 `chat --interactive`。目标是让用户在终端交互时看到模型正在生成、正在调用哪些工具、工具参数是什么、工具返回了什么摘要。

本版本不实现 MCP 接入、API SSE、Web UI 流式、`docs summarize` 流式，也不重做审批系统。MCP 顺延到后续版本；v0.3 先把现有内置工具的可见性和交互体验补齐。

## 背景

v0.2 已经具备 DeepAgents Runtime、交互式 chat、内置工具注册、LangGraph interrupt 审批和 SQLite checkpoint。当前缺口是：

- `chat --interactive` 调用 `AgentSession.run()` 后等待最终结果，期间没有 token 增量。
- Agent 调用 `search_web`、`read_document`、`run_command`、`run_python`、`write_report` 时，用户看不到调用开始、参数和结果摘要。
- 需要审批的工具会出现确认提示，但确认前后的整体执行过程仍像黑盒。

v0.3 要解决这个黑盒体验，但保持已有同步入口稳定。

## 范围

包含：

- 新增稳定的 `StreamEvent` 数据类，作为 Runtime 到 CLI 的事件协议。
- `DeepAgentSession` 新增 `stream(prompt)`。
- `chat --interactive` 在真实模型模式下优先使用 `stream(prompt)`。
- CLI 用 Rich 渲染 token、工具调用开始、工具调用结果、审批提示和最终换行。
- 流式路径继续支持现有 interrupt 审批。
- 保留 `DeepAgentSession.run()` 和 `DeepAgentsRuntime.run()` 的旧行为。
- 增加 fake agent / fake session 测试，不依赖真实模型。

不包含：

- `chat "prompt"` 单次命令流式输出。
- `docs summarize` 流式输出。
- `POST /runs/stream` 或其他 SSE API。
- Web UI、桌面 UI。
- MCP Client Adapter。
- 新的权限策略、沙箱、风险分级系统。
- 长任务恢复 UI。

## 方案

采用“Runtime 事件流 + 交互式 CLI 渲染”的窄切片方案。

`DeepAgentSession.stream(prompt)` 使用底层 `agent.stream(...)` 获取 LangGraph / DeepAgents 事件，并将不同形态的底层事件转换为 easy-claw 自己的 `StreamEvent`。CLI 只依赖 `StreamEvent`，不直接理解 LangGraph 的内部事件结构。

同步调用保持原样：

- `DeepAgentsRuntime.run(request)` 继续打开 session 并调用 `session.run(request.prompt)`。
- `DeepAgentSession.run(prompt)` 暂时继续使用 `_invoke_with_approval()`。
- 后续版本如果底层事件稳定，可以再把 `run()` 改成收集 `stream()` 的 `done` 事件。

这样可以把 v0.3 的风险限制在交互式 CLI，不影响文档工作流、API 和现有测试。

## 事件模型

新增：

```python
@dataclass(frozen=True)
class StreamEvent:
    type: str
    content: str = ""
    tool_name: str | None = None
    tool_args: object | None = None
    tool_result: object | None = None
    thread_id: str | None = None
```

事件类型：

- `token`：模型增量文本。`content` 为本次新增文本。
- `tool_call_start`：工具调用开始。`tool_name` 和 `tool_args` 有值。
- `tool_call_result`：工具返回。`tool_name` 和 `tool_result` 有值，`content` 可存放摘要文本。
- `approval_required`：底层返回 interrupt，CLI 可在审批提示前插入说明。
- `done`：一轮结束。`content` 为最终完整回复，`thread_id` 为当前 thread。

`type` 先用字符串而不是 Enum，保持测试和后续事件扩展简单。

## 底层事件转换

`DeepAgentSession.stream(prompt)` 的流程：

1. 调用底层 `agent.stream({"messages": [...]}, config, stream_mode="messages")`。
2. 遍历底层事件，尽量从消息对象中识别：
   - assistant 文本增量 -> `token`
   - tool call chunk / tool call message -> `tool_call_start`
   - tool result message -> `tool_call_result`
3. 如果底层 stream 返回 interrupt，yield `approval_required`，调用现有 reviewer，随后用 `Command(resume={"decisions": decisions})` 继续 stream。
4. 维护一份完整回答文本，结束时 yield `done`。

由于 deepagents / LangGraph 事件形态可能随版本变化，转换逻辑要集中放在 `runtime.py` 的小函数里，例如：

- `_stream_with_approval(...)`
- `_events_from_stream_item(...)`
- `_tool_call_events_from_message(...)`
- `_tool_result_event_from_message(...)`

如果遇到未知事件，默认忽略，不让 CLI 崩溃。

## CLI 渲染

`_run_interactive_chat()` 已经在真实模型模式下复用一个 `DeepAgentSession`。v0.3 在进入 `_run_interactive_loop()` 时传入两种能力之一：

- `stream_turn: Callable[[str], Iterable[StreamEvent]] | None`
- `run_turn: Callable[[str], AgentResult]`

循环行为：

1. 如果 `stream_turn` 存在，调用 `_render_streaming_turn(stream_turn(prompt))`。
2. 如果不存在，继续调用旧的 `run_turn(prompt)`。
3. 每轮结束后记录 `agent_run` 审计日志。

渲染规则：

- `token`：用 `console.print(event.content, end="")` 或等价方式连续输出。
- `tool_call_start`：先补一个换行，再显示 Rich `Panel`，标题为 `Tool call: <name>`，内容为参数的格式化摘要。
- `tool_call_result`：显示 Rich `Panel`，标题为 `Tool result: <name>`，内容为结果摘要。
- `approval_required`：显示一行 `Tool execution requires approval`，随后交给现有 `ConsoleApprovalReviewer` 输出具体确认问题。
- `done`：确保终端以换行结束，不重复打印完整回复。

工具结果摘要最多显示固定长度，例如 1200 字符。被截断时追加 `[truncated]`。完整工具结果仍然由 Agent 接收，不因为 CLI 摘要截断而影响推理。

## 审批行为

现有 `ConsoleApprovalReviewer` 继续负责读取 interrupt payload、展示工具名和参数、询问 `Allow this action? [y/N]`。

流式路径只改变两点：

- 在发现 interrupt 时先 yield `approval_required`，让 CLI 能把流式文本和审批提示隔开。
- reviewer 返回 decisions 后，使用 `Command(resume={"decisions": decisions})` 恢复底层 stream。

拒绝审批时的行为沿用现有 LangGraph / deepagents 处理结果，不新增特殊分支。

## 错误处理

- 如果底层 `agent.stream` 不存在，CLI 自动退回同步 `run()`。
- 如果底层事件无法识别，忽略该事件。
- 如果 stream 中抛出异常，异常按现有 Typer 行为暴露，不在 v0.3 新增恢复机制。
- 工具结果面板渲染失败时，退回 `str(value)`。
- `dry-run` 保持旧行为，不模拟 token 流。

## 测试

新增或调整测试：

- `StreamEvent` 可创建 token、工具开始、工具结果、done 事件。
- fake streaming agent 的 assistant 文本事件能转换为 `token` 和 `done`。
- fake streaming agent 的工具调用事件能转换为 `tool_call_start`。
- fake streaming agent 的工具结果事件能转换为 `tool_call_result`。
- fake streaming agent 返回 interrupt 时，会调用 `StaticApprovalReviewer` 并用 `Command(resume=...)` 继续。
- `DeepAgentSession.run()` 旧测试继续通过。
- `chat --interactive` 在 fake session 提供 `stream()` 时使用流式渲染，并记录 `agent_run`。
- `chat --interactive --dry-run` 继续走旧 dry-run 输出。

验证命令：

```powershell
uv run pytest tests/test_agent_runtime.py tests/test_cli.py -q
uv run pytest -q
uv run ruff check .
```

- `uv run easy-claw chat --interactive` 在真实模型模式下可以边生成边显示。
- Agent 调用内置工具时，终端能显示工具名、参数和结果摘要。
