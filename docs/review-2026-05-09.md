# easy-claw v0.5.0 全量审查报告

> 审查日期：2026-05-09 | 聚焦：可用性、用户交互体验、架构可扩展性、安全性

---

## 一、可用性

### 做得好

- **一键启动脚本** (`scripts/start.ps1`) 涵盖 `uv sync`、`init-db` 和交互模式，降低上手成本
- **MCP auto 模式**实现优雅降级——缺少配置时不会崩溃，只输出警告
- **`/doctor` 诊断命令**覆盖数据库、模型、密钥（脱敏显示为 `***xxxx`）、MCP、浏览器等全部配置项
- **`/status` 实时状态**展示当前会话的 token 用量、轮次、模型等核心指标
- 丰富的**斜杠命令**：`/help`、`/save`、`/resume`、`/workspace`、`/model`、`/sessions` 等

### 问题列表

#### U-01: 空行输入无反馈

- **文件**: `src/easy_claw/cli_interactive.py:172`
- **现象**: 空行直接 `continue`，用户完全不知道发生了什么
- **影响**: 用户按 Enter 后界面无任何反应，可能怀疑程序卡死
- **建议**: 至少打印一个淡色提示（如 "输入为空，已跳过"）；或者直接不做处理更自然。当前 "空行会跳过" 只在启动 banner 中提了一次

#### U-02: 默认审批模式过于宽松

- **文件**: `src/easy_claw/config.py:87`
- **现象**: `approval_mode` 默认值为 `"permissive"`，AI 可自由执行任意 PowerShell 和 Python 代码
- **影响**: 新用户可能不理解工具的执行能力边界，存在误操作风险
- **建议**: 首次启动展示醒目的安全提示；或考虑默认值改为 `"balanced"`

#### U-03: Web 端与 CLI 功能分裂

- **文件**: `src/easy_claw/api/static/index.html:489`
- **现象**: `/workspace`、`/model`、`/doctor`、`/delete-session` 在 Web 端返回静态提示 "目前仅支持 CLI"
- **影响**: 用户从 CLI 切换到 Web 时发现功能缺失，体验不连贯
- **建议**: 这些命令对应的后端 API 已经存在，前端只需对接即可

#### U-04: Web 端刷新即丢失全部消息

- **文件**: `src/easy_claw/api/static/index.html`
- **现象**: 消息历史仅保存在客户端 DOM 中，无服务端加载历史对话的能力
- **影响**: 刷新页面后所有对话消失
- **建议**: 利用 LangGraph 的 checkpoint DB 加载历史消息，在 WebSocket 连接建立后回放

#### U-05: 工作区与模型切换不持久化

- **文件**: `src/easy_claw/cli_interactive.py:108-121`
- **现象**: `/workspace` 和 `/model` 切换仅在内存中（`dataclasses.replace`），重启后恢复 `.env` 默认值
- **影响**: 用户可能困惑为何上次的切换没有保留
- **建议**: 将切换后的值写入会话记录（`sessions` 表已有 `workspace_path` 和 `model` 字段），恢复会话时自动加载

## 二、用户交互体验

### 做得好

- **流式 token 输出** (`_render_streaming_turn`) 实现打字效果，tool call 以 Rich Panel 内嵌，蓝色=调用、绿色=结果，视觉效果清晰
- **"思考中" spinner** 在等待模型首个 token 期间持续显示，避免用户猜测是否卡死
- **Web UI 的 tool panel** 可折叠，自动 HTML 转义防 XSS，结果展示友好
- **自动重连** WebSocket 断线后 2 秒自动重连
- **键盘快捷键**: Enter 发送、Shift+Enter 换行、Escape 关闭弹窗

### 问题列表

#### UX-01: Tool Call 结果截断限值过小

- **文件**: `src/easy_claw/cli_interactive.py:28`
- **现象**: `STREAM_PANEL_VALUE_LIMIT = 1200` 字符，远低于后端 20000 的实际限制
- **影响**: 返回 JSON 的工具调用（如 MCP 工具）可能只展示开头一小段，用户看不到完整结果
- **建议**: 提升到 4000-6000 字符，或支持按键展开完整结果

#### UX-02: Web 端自动批准无用户感知

- **文件**: `src/easy_claw/api/main.py:212`
- **现象**: `StaticApprovalReviewer(approve=True)` 自动批准所有工具调用，前端仅在状态行显示一行 "工具执行需要确认（已自动批准）"
- **影响**: 用户可能完全不知道 AI 正在执行本地命令
- **建议**: 在消息区插入明显的批准提示条；或提供 Web 端的确认交互

#### UX-03: 代码块无语法高亮

- **文件**: `src/easy_claw/cli_interactive.py:_format_stream_value`
- **现象**: Rich 面板中的工具结果都是纯文本，未使用 Rich 的 `Syntax` 组件
- **影响**: Python/JSON 输出可读性差
- **建议**: 对常见格式（JSON、Python traceback、diff）自动检测并使用 `Syntax` 渲染

#### UX-04: 恢复会话不显示历史消息

- **文件**: `src/easy_claw/cli_interactive.py:137-139`
- **现象**: `/resume` 后 `conversation.clear()` 清除客户端记录，之前的对话不在终端显示
- **影响**: 用户看不到之前的上下文，不知道会话进行到哪一步
- **建议**: 从 LangGraph checkpoint 中读取历史消息并摘要显示（至少显示最近几轮）

#### UX-05: 错误恢复建议不足

- **文件**: `src/easy_claw/agent/runtime.py:368`
- **现象**: `_format_agent_runtime_error` 返回简短的 "Agent 执行失败：{message}"，无操作建议
- **影响**: 用户看到错误不知道下一步该做什么
- **建议**: 对常见错误类型（如 API key 无效、网络超时、速率限制）提供针对性的操作建议

## 三、架构与可扩展性

### 做得好

- **清晰的包结构**: `agent/`（运行时 + 中间件）、`tools/`（核心 + 浏览器 + MCP）、`api/`（REST + WebSocket）、`storage/`（仓库模式）、顶层模块职责分明
- **`ToolBundle` 模式** (`agent/types.py:22`) 将工具列表、清理回调、审批策略打包，添加新工具类别只需返回新的 `ToolBundle`
- **不可变配置** `AppConfig` 是 frozen dataclass，修改用 `dataclasses.replace()`，避免隐式状态变更
- **后台事件循环** `_BackgroundEventLoop` (`tools/base.py`) 优雅解决了跨线程 asyncio 冲突
- **多源技能系统** `resolve_skill_sources` (`skills.py`) 从 builtin、user home、project 多层目录收集 skill，优先级明确
- **MCP 多服务容错** `_async_init_mcp` 支持逐个服务加载，一个失败不影响其他
- **`uv` + `hatchling`** 现代化项目管理，依赖锁文件确保可重现性

### 问题列表

#### ARCH-01: `_count_mcp_servers` 和 `_mcp_status` 代码重复

- **文件**: `src/easy_claw/cli_views.py:236-258` 和 `src/easy_claw/api/main.py:47-69`
- **现象**: 两个模块各自实现了几乎完全相同的函数
- **影响**: 修改 MCP 状态逻辑需要同步两处，容易遗漏
- **建议**: 抽取到 `src/easy_claw/tools/mcp.py` 或新建 `src/easy_claw/status.py`，CLI 和 API 共用

#### ARCH-02: 运行时内部延迟导入

- **文件**: `src/easy_claw/agent/runtime.py:112-113`
- **现象**: `from deepagents import create_deep_agent` 和 `from langgraph.checkpoint.sqlite import SqliteSaver` 在 `open_session()` 方法体内导入
- **影响**: 依赖关系不透明，IDE 无法静态分析，测试需要 mock 这些模块
- **建议**: 提升到模块级别的 try/except 导入，或至少抽到 `_build_agent()` 私有函数中

#### ARCH-03: Web 前端是单体 HTML 文件

- **文件**: `src/easy_claw/api/static/index.html`（668 行）
- **现象**: 全部 CSS + JS 内联在单个文件中
- **影响**: 当前功能量尚可，但继续增加功能时将迅速失控
- **建议**: 拆分为 `index.html` + `style.css` + `app.js`；或引入轻量框架（如 Alpine.js / htmx）

#### ARCH-04: 缺少 LLM 提供商抽象层

- **文件**: `src/easy_claw/agent/runtime.py:208-215`
- **现象**: `_build_chat_model` 硬编码使用 `langchain_openai.ChatOpenAI`，所有模型必须兼容 OpenAI API 格式
- **影响**: 如果要支持原生 DeepSeek SDK、Anthropic SDK 等非 OpenAI 兼容的 API，需要大幅重构
- **建议**: 定义 `ModelProvider` 协议/抽象，支持通过配置切换不同的 SDK 后端

#### ARCH-05: 缺少插件/扩展机制

- **文件**: `src/easy_claw/agent/toolset.py:15-38`
- **现象**: 工具、技能、搜索后端的注册路径都是硬编码的
- **影响**: 第三方无法通过安装包来添加新工具或技能
- **建议**: 设计注册表模式（Registry Pattern），允许外部通过 entry points 或配置文件注册自定义工具

#### ARCH-06: `AgentRequest.config` 允许 None 但实际不接受

- **文件**: `src/easy_claw/agent/runtime.py:21,98-99`
- **现象**: `config: AppConfig | None` 但在 `open_session()` 中立即断言非空才抛异常
- **影响**: 类型设计上允诺了不存在的状态，增加了不必要的 None 检查路径
- **建议**: 移除 `| None`，让调用方必须传入有效配置

#### ARCH-07: 测试覆盖缺口

- **现象**: 缺少对 `cli_interactive.py`、`cli_slash.py`、`cli_views.py` 的直接单元测试
- **影响**: 核心交互循环和流式渲染没有自动化验证，回归风险高
- **建议**: 补充以下测试：
  - `test_cli_interactive.py` — 流式渲染事件转文本、空输入、EOF 处理
  - `test_cli_slash.py` — 命令分发、别名、错误命令
  - `test_cli_views.py` — 状态展示、会话列表、启动横幅

## 四、安全性

### 当前水平

作为本地优先（默认 `127.0.0.1`）的个人工具，当前安全水平可接受。但以下问题值得关注：

### 问题列表

#### SEC-01: PowerShell 无沙箱执行

- **文件**: `src/easy_claw/tools/commands.py:28-35`
- **现象**: `run_command` 直接通过 PowerShell 执行 AI 生成的命令，虽有 `-NoProfile` 但无任何沙箱限制
- **风险**: 在 `permissive` 审批模式下，AI 可执行任意系统命令
- **建议**: `execution_mode` 配置已预留 `sandbox` 选项但未实现；考虑使用 Windows Sandbox 或 Docker 隔离

#### SEC-02: Python 代码无沙箱执行

- **文件**: `src/easy_claw/tools/python_runner.py:9-27`
- **现象**: `run_python_code` 将 AI 生成的代码写入临时 `.py` 文件并用系统 Python 执行
- **风险**: 代码可访问文件系统、网络、进程，与 `run_command` 同等风险
- **建议**: 同 SEC-01，实现沙箱模式

#### SEC-03: 路径遍历风险

- **文件**: `src/easy_claw/tools/documents.py:38-46` 和 `src/easy_claw/workspace.py:10-18`
- **现象**: `resolve_user_path` 允许读取绝对路径，`read_document` 对工作区外文件仅标记 `outside_workspace=True` 但仍然读取
- **风险**: AI 可被诱导读取系统敏感文件（如 `C:\Windows\System32\config\SAM`）
- **建议**: `balanced` 和 `strict` 审批模式下，对工作区外路径应要求额外确认

#### SEC-04: WebSocket 无认证

- **文件**: `src/easy_claw/api/main.py:187-189`
- **现象**: `/ws/chat` 端点无任何认证机制
- **风险**: 局域网内任何客户端都可以连接并控制 Agent
- **建议**: 当前默认监听 `127.0.0.1`，风险可控。但若用户改为 `0.0.0.0`，需增加 token 认证或 CORS 限制

#### SEC-05: API 无速率限制

- **文件**: `src/easy_claw/api/main.py`
- **现象**: 所有 REST 端点和 WebSocket 无 rate limiting
- **风险**: 可能被滥用导致 API 费用失控（LLM 调用是付费的）
- **建议**: 至少对聊天相关的端点添加速率限制

## 五、总结

### 各维度评分

| 维度 | 评分 (1-10) | 关键原因 |
|---|---|---|
| 可用性 | 7 | 启动/诊断流程完善，但 CLI/Web 功能分裂，空行交互缺失 |
| 用户体验 | 7.5 | 流式渲染优秀，但截断限值过低，无历史回显，无语法高亮 |
| 架构/扩展性 | 7.5 | 模块划分清晰，但存在代码重复，缺少抽象层和插件机制 |
| 安全性 | 6.5 | 本地使用场景可接受，但无沙箱、无认证、无速率限制 |

### 优先修复建议（按影响排序）

1. **ARCH-01: 消除代码重复** — 低风险高收益，一行动作减少维护负担
2. **UX-01: 提高 `STREAM_PANEL_VALUE_LIMIT`** — 一行改动改善大量工具调用的展示体验
3. **UX-04 / U-04: Web 端历史消息加载** — 对 Web 体验提升最大的一块
4. **U-01: 空行输入反馈** — 低成本消除用户困惑
5. **SEC-01 / SEC-02: 首次启动安全提示** — 降低 permissive 模式的风险认知差距
6. **U-03: 统一 CLI/Web 命令支持** — 消除功能分裂
7. **ARCH-03: Web 前端拆分** — 在功能膨胀前建立更好的代码组织
8. **ARCH-07: 补充交互层测试** — 降低回归风险
